# Qiangua Dashboard

千瓜 Excel 上传、分析和看板服务。每天上传千瓜导出的榜单文件后，系统会完成文件校验、数据入库、AI 洞察分析，并生成数据看板、洞察日报和任务详情页。

## 页面入口

启动服务后打开：

```bash
python qiangua_upload_server.py --port 8090
```

常用页面：

| 页面 | 地址 | 用途 |
| --- | --- | --- |
| 上传任务中心 | `http://localhost:8090/` | 上传千瓜 Excel/压缩包，查看最近任务 |
| 数据看板 | `http://localhost:8090/dashboard/YYYY-MM-DD` | 查看笔记、热词、话题等数据指标 |
| 洞察日报 | `http://localhost:8090/insights/YYYY-MM-DD` | 查看 AI 生成的趋势、机会和选题建议 |
| 任务详情 | `http://localhost:8090/runs/{id}` | 查看单次上传的阶段、错误、数据质量和文件明细 |

## 支持的文件

可以上传 `.xls`、`.xlsx`，也可以上传包含 Excel 的 `.zip`、`.tar`、`.tar.gz`、`.tgz`、`.gz`。

文件名需要能识别千瓜榜单类型：

| 榜单类型 | 文件名关键词 |
| --- | --- |
| 实时笔记榜单 | `实时笔记榜单` |
| 6 小时实时笔记榜单 | `实时笔记榜单` + `6小时` |
| 低粉笔记榜单 | `低粉笔记榜单` |
| 商业笔记榜单 | `商业笔记榜单` |
| 热词增量榜 | `热词增量榜` |
| 热词总量榜 | `热词总量榜` |
| 热门话题增量榜 | `热门话题增量榜` |
| 热门话题总量榜 | `热门话题总量榜` |
| 流量扶持榜 | `流量扶持榜` |

## 上传校验

上传后会先做文件级校验：

- 识别文件类型。
- 检查 Excel 是否能读取。
- 检查是否为空表。
- 检查关键字段是否存在。
- 对每个文件记录状态、类型、入库行数和错误原因。

关键字段要求：

| 类型 | 必要字段 |
| --- | --- |
| 笔记类榜单 | `排名`、`笔记标题`、`笔记链接`、`互动量` |
| 热词类榜单 | `排名`、`热搜词名称` |
| 话题类榜单 | `排名`、`话题名称` |

如果一批文件里有部分无效文件，系统会跳过无效文件并继续处理有效文件；如果全部无效，任务会失败，并在任务详情页显示失败原因。

## 任务详情

每次上传都会生成一条任务记录。任务详情页包含：

- 当前任务状态和阶段。
- 上传文件明细。
- 每个文件的识别类型、状态、入库行数、错误原因。
- 数据质量统计：笔记数、热词数、话题数、入库总行数。
- 数据看板和洞察日报入口。

## 环境配置

推荐 Python 3.10+。

安装依赖：

```bash
pip install -r requirements.txt
```

MiniMax API Key 放在：

```text
~/.minimax_key
```

也可以通过环境变量提供：

```bash
set MINIMAX_API_KEY=your_key
```

## 常见问题

| 问题 | 处理 |
| --- | --- |
| 未识别的千瓜文件类型 | 检查文件名是否包含支持的榜单关键词 |
| Excel 读取失败 | 检查文件是否损坏，或是否为真正的 Excel 文件 |
| 缺少必要字段 | 对照上传校验表检查千瓜导出字段是否变化 |
| 看板为空 | 确认任务详情里的文件是否成功入库 |
| 没有洞察日报 | 确认 MiniMax API Key 是否已配置 |
